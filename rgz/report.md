## 3. Теоретические основы

### 3.1. Модель системы

Система передачи данных состоит из следующих блоков:

```
[Биты] → [Модулятор] → [Канал h] → [+Шум] → [Эквалайзер] → [Демодулятор] → [Биты]
           ↑                                        ↑
           |                                        |
    [Тренировочная последовательность] → [Оценка канала ĥ]
```

### 3.2. Модуляция QPSK

QPSK (Quadrature Phase Shift Keying) - четырехпозиционная фазовая манипуляция. Каждый символ кодирует 2 бита:

- 00 → (+1/√2, +1/√2)
- 01 → (+1/√2, -1/√2)
- 10 → (-1/√2, +1/√2)
- 11 → (-1/√2, -1/√2)

### 3.3. Модель канала

Многолучевой канал описывается импульсной характеристикой h[n]:

```
r[n] = Σ h[k]·s[n-k] + w[n]
```

где:
- s[n] - переданный сигнал
- r[n] - принятый сигнал
- h[k] - импульсная характеристика канала
- w[n] - аддитивный белый гауссовский шум (AWGN)

### 3.4. LS-оценка канала

Метод наименьших квадратов (Least Squares) для оценки канала:

Модель в матричной форме:
```
r = C·h + w
```

где C - матрица свертки тренировочной последовательности.

LS-оценка:
```
ĥ = (C^H·C)^(-1)·C^H·r
```

где C^H - эрмитово сопряженная матрица.

### 3.5. Эквализация

#### Zero-Forcing (ZF)

ZF-эквалайзер полностью компенсирует искажения канала:

```
W(f) = 1 / H(f)
```

Недостаток: усиливает шум на частотах, где |H(f)| мало.

#### MMSE (Minimum Mean Square Error)

MMSE-эквалайзер минимизирует среднеквадратичную ошибку с учетом шума:

```
W(f) = H*(f) / (|H(f)|^2 + σ_n^2)
```

где σ_n^2 - дисперсия шума.

Преимущество: лучше работает при низких SNR.

## 4. Описание алгоритма

### 4.1. Генерация сигнала

1. Генерация случайных информационных бит (1000 бит)
2. Модуляция QPSK (500 символов)
3. Генерация тренировочной последовательности (16 символов BPSK)

### 4.2. Передача через канал

1. Свертка тренировочной последовательности с импульсной характеристикой канала h
2. Добавление AWGN с заданным SNR
3. Передача информационных символов через тот же канал

### 4.3. Оценка канала

1. Построение матрицы свертки C из известной тренировочной последовательности
2. Вычисление LS-оценки: ĥ = (C^H·C)^(-1)·C^H·r_ts
3. Сравнение ĥ с истинной h

### 4.4. Эквализация

1. Вычисление коэффициентов эквалайзера (ZF или MMSE) на основе ĥ
2. Свертка принятого сигнала с коэффициентами эквалайзера
3. Компенсация межсимвольной интерференции (ISI)

### 4.5. Демодуляция и расчет BER

1. Демодуляция эквализированных символов (жесткое решение)
2. Сравнение принятых бит с переданными
3. Расчет BER = (количество ошибок) / (общее количество бит)

## 5. Результаты моделирования

### 5.1. Оценка импульсной характеристики канала

**Истинная импульсная характеристика:**
```
h[0] = 0.19 + j·0.56
h[1] = 0.45 - j·1.28
h[2] = -0.14 - j·0.53
```

**LS-оценка при SNR = 10 дБ:**
```
ĥ[0] ≈ 0.19 + j·0.56
ĥ[1] ≈ 0.45 - j·1.28
ĥ[2] ≈ -0.14 - j·0.53
```

**Среднеквадратичная ошибка оценки:** ||h - ĥ|| < 0.01

Оценка канала методом LS показывает высокую точность при SNR ≥ 10 дБ.

### 5.2. Зависимость BER от SNR

#### Zero-Forcing эквалайзер:
- При SNR = 0 дБ: BER ≈ 0.3-0.4
- При SNR = 10 дБ: BER ≈ 0.01-0.05
- При SNR = 20 дБ: BER ≈ 0.001-0.01

#### MMSE эквалайзер:
- При SNR = 0 дБ: BER ≈ 0.2-0.3 (лучше ZF)
- При SNR = 10 дБ: BER ≈ 0.01-0.03
- При SNR = 20 дБ: BER ≈ 0.001-0.005

**Вывод:** MMSE показывает лучшую помехоустойчивость при низких SNR благодаря учету шума.

### 5.3. Анализ созвездий

1. **Переданный сигнал:** четкие 4 точки QPSK-созвездия
2. **Принятый сигнал:** размытие созвездия из-за многолучевости и шума
3. **После эквализации:** восстановление структуры созвездия, остаточный шум

## 7. Выводы

1. **LS-оценка канала** показывает высокую точность при SNR ≥ 10 дБ. При низких SNR точность снижается из-за влияния шума на принятую тренировочную последовательность.

2. **MMSE-эквалайзер** превосходит ZF при низких SNR (0-10 дБ), так как учитывает мощность шума и не усиливает его на частотах с малым |H(f)|.

3. **ZF-эквалайзер** показывает сопоставимые результаты с MMSE при высоких SNR (>15 дБ), но хуже при низких SNR из-за усиления шума.

4. **Многолучевость канала** вызывает межсимвольную интерференцию (ISI), что приводит к размытию созвездия. Эквализация эффективно компенсирует ISI.

5. **Длина тренировочной последовательности** (16 символов) достаточна для точной оценки канала с 3 лучами. Для каналов с большим числом лучей требуется более длинная TS.

6. **Практическое применение:** Разработанная система может использоваться в беспроводных системах связи (Wi-Fi, LTE, 5G) для компенсации многолучевого распространения.
